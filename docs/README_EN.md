Apple Watch's built-in ECG can only tell you "Sinus Rhythm" or "Atrial Fibrillation". Two classes, nothing more. **HeartLens takes this an order of magnitude further.**

From the same Apple Watch ECG recording, HeartLens identifies **5 cardiac abnormality classes** (Myocardial Infarction, Conduction Disturbance, ST/T Changes, Ventricular Hypertrophy, Normal) and **colour-codes your ECG waveform** to show exactly where the AI is looking: red regions indicate high model attention, blue regions are ignored. You don't just get a label; you see *which waveform features, at which time points*, drove the decision. Finally, a locally-running Qwen3.5-4B vision model directly "reads" the annotated ECG image and generates a plain-language clinical interpretation.

Everything runs fully offline. Zero cloud calls, zero data uploads, zero API keys.

> **Why Qwen3.5-4B?** We benchmarked GPT-5.4 (commercial API) against the full Qwen3.5 family (0.8B / 2B / 4B) across 34 test scenarios. In multimodal mode (LLM directly viewing the Grad-CAM heatmap), **Qwen3.5-4B matched GPT-5.4's clinical explanation quality with zero hallucinations and 100% completeness**, while running 3x faster. The entire model is 3.4 GB and runs on a consumer GPU or even CPU-only.

![Architecture](../assets/architecture.png)

## What It Does

1. **Reads Apple Watch ECGs** — Parses CSV exports from the iPhone Health app (512 Hz single-lead)
2. **5-class cardiac screening** — CNN-LSTM classifier detects Myocardial Infarction, Conduction Disturbance, ST/T Changes, Ventricular Hypertrophy, and Normal (Apple Watch's built-in offers only 2 classes)
3. **Colour-coded waveform visualisation** — Grad-CAM heatmaps overlay red/blue attention directly on your ECG trace, making every prediction visually auditable
4. **Local Qwen3.5 interpretation** — On-device Qwen3.5-4B multimodal model "sees" the annotated ECG image and generates a clinical report, no external API required
5. **Fully offline** — From recording to report, all computation stays on your machine

## Demo

### Your ECG Gets Colour-Coded

![Grad-CAM Example](../assets/gradcam_example.png)

> This ECG comes from the PTB-XL clinical database (**standard 12-lead** recording, showing Lead I here). **Red/orange = high attention**, **blue = ignored**. This example shows Conduction Disturbance (CD, 97% confidence). Notice the model does **not** attend to the tallest R-wave peaks (blue); instead, attention concentrates on the secondary deflections following each R-wave (red/orange). This aligns with clinical reasoning for conduction disturbance: the abnormality lies not in R-wave amplitude but in QRS morphology changes (widening, notching, secondary deflections) that distinguish normal conduction from block. The lower panel shows raw attention magnitude, with periodic peaks corresponding to the abnormal morphology region in each heartbeat cycle.

### Real Apple Watch ECG Analysis

![Apple Watch Demo](../assets/apple_watch_demo.png)

> Apple Watch captures only a **single lead** (equivalent to Lead I), carrying far less information than a hospital 12-lead ECG. Yet HeartLens's single-lead model (macro AUC 0.832) still extracts sufficient diagnostic signal. Above is a real ECG exported from a team member's Apple Watch (recorded March 2026). The waveform is clean, predominantly blue with no concentrated red regions, indicating the model found no abnormalities. Final verdict: Sinus Rhythm (Normal), 91% confidence. This is what a healthy heart looks like through HeartLens.

## Quick Start

Three steps, no GPU required:

```bash
# 1. Clone and set up
git clone https://github.com/patr1ckzhu/HeartLens-AppleWatch.git
cd HeartLens-AppleWatch
conda env create -f environment.yml && conda activate heartlens

# 2. Pull the local LLM (Qwen3.5-4B, 3.4 GB, one-time download)
curl -fsSL https://ollama.com/install.sh | sh
ollama pull qwen3.5:4b

# 3. Launch
python demo/app.py
```

Open your browser, upload an Apple Watch ECG CSV, and within seconds you'll see: classification results + attention heatmap + a clinical interpretation generated by Qwen3.5.

### How to Export ECG from Apple Watch

Apple Watch ECG data cannot be exported per-recording; you need to export all health data at once via the Health app:

1. Open the **ECG** app on your Apple Watch, hold your finger on the Digital Crown for 30 seconds to record
2. On iPhone, open **Health** → tap your **profile picture** (top right) → scroll to bottom → **Export All Health Data**
3. Wait a few minutes, then choose "Save to Files" or AirDrop to your computer
4. Unzip the export. ECG files are in `apple_health_export/electrocardiograms/`:

```
apple_health_export/
  └── electrocardiograms/
        ├── ecg_2024-01-15.csv
        ├── ecg_2024-03-20.csv
        └── ...
```

Upload any of these CSV files to HeartLens.

## Why Qwen3.5-4B + Multimodal?

The conventional approach converts AI analysis results into text descriptions before feeding them to an LLM. But smaller models fed text-only inputs tend to **hallucinate** (fabricate non-existent abnormalities). We found a better approach:

**Feed the colour-coded ECG image directly to Qwen3.5's vision capability.**

When the LLM can *see* the actual ECG waveform and attention distribution, it stays grounded in visual evidence instead of inventing findings.

| Mode | GPT-5.4 | Qwen3.5-4B | Qwen3.5-2B | Qwen3.5-0.8B |
|------|---------|------------|------------|---------------|
| Text input (hallucination rate) | **0%** | 15% | 24% | 15% |
| Image input (hallucination rate) | **0%** | **0%** | **0%** | **0%** |
| Average latency | 7.4s | **2.7s** | 2.0s | 1.2s |
| Requires internet? | Yes | **No** | **No** | **No** |

**Conclusion: Qwen3.5-4B + multimodal = commercial API quality + on-device privacy.**

### On-Device Deployment Roadmap

Our experimental results support a tiered on-device deployment strategy:

| Tier | Model | Size | Use Case |
|------|-------|------|----------|
| Watch / Wearable | Qwen3.5-0.8B | 1.0 GB | Real-time heart rate monitoring + anomaly alerts |
| Phone | Qwen3.5-4B | 3.4 GB | Full ECG analysis + colour visualisation + interpretation |
| Laptop / Clinic | Qwen3.5-4B / 9B | 3.4-7 GB | Multi-lead analysis + detailed clinical reporting |

All models from 0.8B to 4B achieve zero hallucination in multimodal mode. Model selection depends on your hardware and latency requirements.

## Technical Architecture

```
Apple Watch ECG (512 Hz CSV)
  → Resample to 500 Hz + bandpass filter (0.5-40 Hz) + z-score normalisation
  → CNN-LSTM classifier
      Residual 1D convolution (SE attention) → Bidirectional LSTM → 5-class diagnostic probabilities
  → Grad-CAM attention heatmap (ECG waveform colour-coding)
  → Qwen3.5-4B multimodal inference
      Input: annotated ECG image + diagnostic probabilities → Output: natural language clinical interpretation
```

### Ablation Study

| Model | Macro AUC | Parameters |
|-------|-----------|------------|
| Random Forest (baseline) | 0.861 | — |
| LSTM-only | 0.908 | 0.7M |
| CNN-only | 0.912 | 1.2M |
| **CNN-LSTM (this project)** | **0.914** | **1.5M** |
| CNN-Transformer | 0.913 | 3.3M |

The CNN encoder contributes the majority of classification performance. LSTM and Transformer show negligible differences on fixed-length 10-second recordings. We chose CNN-LSTM for its smallest parameter count, most stable training, and best Grad-CAM compatibility.
